# Check Follow Status

## Endpoint

**URL:** `/:projectId/users/:userId/follow`

**Method:** `GET`

**Authentication Required:** Yes (Access Token in Authorization Header)

---

## Description

Check whether the authenticated user is following a specific user.

---

## Request

### Path Parameters

| Parameter | Type     | Required | Description                      |
| --------- | -------- | -------- | -------------------------------- |
| `userId`  | `string` | Yes      | ID of the user to check against. |

### Headers

| Header          | Type     | Required | Description                      |
| --------------- | -------- | -------- | -------------------------------- |
| `Authorization` | `string` | Yes      | Bearer token for authentication. |

### Body Parameters

_None_

#### Example Request

```http
GET /proj1234/users/abcd123/follow
Authorization: Bearer <ACCESS_TOKEN>
```

---

## Response

### Success Response (200 OK)

```json
{
  "isFollowing": true
}
```

or

```json
{
  "isFollowing": false
}
```

### Error Responses

#### Server Error (500 Internal Server Error)

```json
{
  "error": "Internal server error.",
  "code": "follow/server-error",
  "details": "<Error message>"
}
```

---

## Notes

- This endpoint checks if a follow relationship exists between the authenticated user and the target user.
- Always returns a 200 response with a boolean flag `isFollowing`.
